# Dream Talker 更新路线图 (Roadmap)

本路线图旨在提升 Dream Talker 的用户体验，增强其作为梦话检测工具的实用性和趣味性。

## Phase 1: 基础体验增强 (Basic Experience)
目标：完善核心交互，让用户能更好得查看和管理记录。

- [ ] **音频波形可视化 (Waveform Visualization)**
    - **描述**: 在播放器或记录卡片上显示音频波形，直观展示声音起伏。
    - **功能**: 允许用户通过点击波形跳转播放进度。
    - **技术方案**: 使用 `wavesurfer.js` 或类似库在前端渲染。

- [ ] **手动修正识别文本 (Manual Text Correction)**
    - **描述**: STT 经常会把模糊的梦话识别错误，允许用户手动编辑 STT 自动生成的转录文本。
    - **功能**: 添加“编辑”按钮，支持修改并保存到数据库。

- [ ] **动态配置面板 (Settings Panel)**
    - **描述**: 目前参数配置依赖文件，不够灵活。
    - **功能**: 在 Web UI 提供设置页面，支持调整 VAD 阈值（灵敏度）、STT 语言偏好等，配置持久化保存。

## Phase 2: AI 智能化 (AI & Insights)
目标：利用 AI 能力深度挖掘数据价值，增加趣味性。

- [ ] **LLM 梦境日报 (Dream Report)**
    - **描述**: 每天早上醒来，能看到昨晚梦话的“智能总结”。
    - **功能**: 接入 LLM (DeepSeek/ChatGPT/Local LLM)，生成梦话摘要，或分析梦话反映的情绪/精神状态（娱乐性质）。

- [ ] **智能/手动标签系统 (Tagging System)**
    - **描述**: 很多梦话其实是“非语言声音”。
    - **功能**: 允许用户手动打标签（如“笑声”、“哭声”、“外语”）；后期尝试使用音频分类模型自动打标签。

- [ ] **梦境绘图 (Dream Visualization)**
    - **描述**: 将听觉转化为视觉。
    - **功能**: 提取梦话关键词，调用文生图 API (Stable Diffusion/Midjourney) 生成一张“梦境快照”。

## Phase 3: 系统与运维 (System & Ops)
目标：提高系统稳定性和可维护性。

- [ ] **存储空间管理 (Storage Management)**
    - **描述**: 长期录音会占用大量磁盘。
    - **功能**: 仪表盘显示磁盘占用情况；提供自动清理策略（如：自动删除 30 天前的非收藏录音）。

- [ ] **移动端适配优化 (PWA/Mobile)**
    - **描述**: 很多用户会把旧手机放在床头作为检测设备。
    - **功能**: 优化 CSS 样式适配手机屏幕；支持 PWA，使其像原生应用一样运行。

- [ ] **消息推送 (Notifications)**
    - **描述**: 即时了解检测情况。
    - **功能**: 支持 Webhook (钉钉/微信/Telegram) 推送；支持“勿扰模式”（仅生成报告不实时推送）。
